Scraping data from Scoresheets

The model is designed to pull as much information which is standard across most / all scoresheets.

The fields which all scoresheets have are contest details (name, date, organization, location), quartet name , results (total score, song names, category scores), and judge details (name, category).

Some fields are not included in all scoresheets but are captured when possible such as quartet rank and quartet location / region. Due low coverage and inconsistency in naming standards, quartet member names are not captured.

Collecting and collating scores from multiple contests, across multiple oranizations, over an extended period of time comes with a number of challenges. Hopefully understanding these challenges and how it is handled on Thinking Barbershop will explain why you might see some slight differences on here to what you would expect.


ACCESS TO SCORESHEETS

Across BHS and Alliance organizations there has been no single database of scores to date. There are thousands of scoresheets of quartet results, so the choice was made early to use a model to parse the scores and populate a database. This does mean that we cannot manually enter scores from any other source, limiting us to requiring the raw scoresheet PDFs for all competitions.

With each organization being run seperately, the quality and availability of contest scoresheets over time can vary. With some organizations only having a couple of years of scoresheets available, and others having decades, the broader analysis will tend to be more reliable and weighted towards those organizations with the most historical scoresheets publicly available.

Additionally there are cases when a quartet will be assessed for a score (competing out of region, performing for assessment only etc...) and not included on any scoresheet. While their score may be valid for entry to other contests, it will not be part of the historical data on this site.


SCORESHEET VARIABILITY

Formatting and completeness of information on scoresheets has changed over the years. Even now, between different organizations, there is no standard scoresheet template.

The model is great but it is not perfect. As historical scoresheets have been added, often the limiting factor is when the model struggles to parse an old structure, so it needs to be manually trained to handle the new (old) format.

Other factors can vary too, like whether all quartets on a given scoresheet are all competing or not. This is particularly challenging with BHS, as often you will have Out of Region quartets, and champion quartets competing to qualify for International. As it currently stands there is no indicator in our database whether a quartet on a scoresheet was competing for a score or not. This is noted as a challenge to resolve in the future but is low on the priority list. For official contest champions please refer to the relevant organization / region official website.


SONG & QUARTET NAME VARIABILITY

Historically most organizations have relied on manual entry from quartet representatives for Quartet Name and Song Names. This means that a quartet can sing the same song across multiple contests and for it to have a different title. Similarly there have been cases where a quartet name will vary slightly between contests.

The data collection philosophy has been to extract exactly what is on the scoresheet, meaning all of these inaccuracies are in the database. To account for this, some standardization has been done to align names which are for the same quartet/song but with different spelling.


ALIGNING ORGANIZATIONS

Each organization operates in isolation, so there are nuances and overlaps which affect the data.

Contest names and structures - Regional, District, International & National streams, Gender categories, qualification structures and rules. All of these vary between the different organizations. To account for this, a broadly universal contest structure has been created, with variations all having unique contest ids. The naming and structure of this has been based on public information about each organization, so there are no doubt nuances which are not captured. If you are interested in helping, we are looking for supporters from each organization to improve the current structure.

Top Shelf is a popular quartet name! There has been a Top Shelf quartet in BHA, JAC, SWD, NSC, and MAD. Without intervention, all of these quartets would be grouped together making it difficult to see individal group progress and achievements. To solve this, a model is run when parsing a new scoresheet to see whether a given quartet name matches an existing quartet based on contest location, contest type (regional, national, international), and overlapping song names. This helps but is not perfect. If you see multiple quartets overlapping, send a screenshot to support@thinkingbarbershop.com and it will be adjusted as soon as possible.


Typically the data work falls into 3 phases

Maintentnace

Maintenance involves processing new contests as they are completed, and fixing any errors.

Backdating

Backdating is sourcing and processing historical scoresheets across each organization, while keeping up to date with new contests.

Expanding

Expanding is the process of introducing new data, such as chorus competitions, or organizations outside the current scope. This is more involved as it may require database restructuring or a new parsing model.

The plan for 2024 is to continue backdating, sourcing historical spreadsheets particularly for European organizations, then moving on to expanding into BHS & Alliance Chorus scores.

With all going to plan, 2025 will look to introduce HI, and then SAI. This order is due to the high similarity in scoresheet structure between HI and BHS.
